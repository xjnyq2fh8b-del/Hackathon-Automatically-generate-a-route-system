# Westlake Now 产品设计方案

## 产品定位

Westlake Now 是面向杭州西湖湖滨“现在就出发”场景的本地路线智能规划 Agent。它不是泛泛生成攻略，而是把用户一句自然语言需求转化成一条可以直接执行、可以继续调整的路线。

核心目标是降低用户在临时出行时的决策成本：不用反复搜索地点、比较距离、判断排队和手动拼路线，系统直接给出下一站、时间轴、地点卡片、地图连线和导航入口。

## 目标用户

- 临时决定去西湖湖滨周边逛一逛的用户
- 有 2-4 小时碎片时间，需要一条马上能走的路线
- 希望同时满足拍照、逛街、吃饭、喝咖啡、少排队、预算等约束的人
- 不想自己在搜索软件里反复筛选和拼接地点的人

## 核心场景

用户输入：

```text
我在湖滨银泰，下午2点到6点，想逛西湖、喝咖啡、吃晚饭，人均150，不想排队
```

系统输出：

- 一条 3-5 个点的可执行路线
- 每一站的推荐理由、停留时间和预算信息
- 总时长、步行距离、交通建议
- 地图 marker、路线连线和高德导航入口
- 支持继续输入“不要咖啡”“预算降到100”“只剩2小时”等二次调整

## 产品流程

```text
用户自然语言输入
  ↓
前端提交 /api/chat-route
  ↓
后端识别意图和约束
  ↓
routePlanner 从 POI 数据中选择地点
  ↓
生成 routeData
  ↓
前端渲染推荐卡片、路线时间轴、地图和下一站
  ↓
用户继续调整路线
```

## 信息架构

### 首页输入

首页聚焦“现在就出发”的自然语言输入，不做复杂筛选表单。用户可以直接说目标、时间、预算、偏好和禁忌。

### 路线结果

路线结果页优先展示用户马上需要做的决策：

- 当前推荐路线
- 下一站
- 总时间
- 总距离
- 预算
- 路线时间轴
- 地点卡片
- 地图

### 二次调整

用户不需要重新开始，可以在当前路线基础上继续输入：

- 不想吃饭
- 不想拍照
- 不喝咖啡
- 预算 100
- 少排队
- 只剩 2 小时

系统会继承旧约束，合并新约束，返回一条新的完整路线。

## 设计原则

1. 路线优先，而不是推荐列表优先。
2. LLM 负责理解语言，routePlanner 负责做路线决策。
3. 每次调整都必须让用户看见路线变化。
4. 前端展示的编号、下一站、地图连线和时间轴必须使用同一套路线路径。
5. 对不可满足的需求要明确解释，不能静默无反应。

## 当前 MVP 边界

当前版本聚焦杭州西湖湖滨周边，不覆盖全城、多日游、酒店票务交易、实时排队预测和复杂 Agent 编排。

当前步行时间主要基于经纬度距离估算，未来可以接入高德实时步行路径规划，让距离和耗时更接近真实出行情况。

## 成功标准

- 用户输入一句话后，可以得到一条可执行路线。
- 用户修改路线需求后，路线会真实变化。
- 路线点位、底部下一站、地图连线、时间轴顺序一致。
- 结果能够解释为什么这样安排，以及修改后发生了什么变化。
- Demo 可以稳定展示完整闭环。
