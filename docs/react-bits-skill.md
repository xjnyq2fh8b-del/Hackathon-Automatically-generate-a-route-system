# React Bits 项目内调用说明

## 定位

`react-bits/` 已安装在项目根目录，来源为 `https://github.com/DavidHDev/react-bits`。

它不是本项目的业务代码，而是前端 UI、动效和组件灵感资源。后续做路线规划 Agent 的前端体验时，可以优先参考这里的组件实现，再迁移或改造成适合本项目的路线规划组件。

## 使用场景

- 路线步骤时间线：例如把多个 POI 按顺序展示成可执行路线。
- POI 卡片交互：例如强调评分、排队风险、价格、停留时长和推荐理由。
- 动态调整反馈：例如用户改预算、时间、偏好后，路线重新生成时的状态动效。
- 地图和路径辅助视觉：例如路线切换、局部高亮、候选方案对比。

## 使用边界

- 只能用于增强路线规划体验，不能把项目变成单纯的动效展示页。
- 迁移组件前，先确认它能帮助用户更清楚地理解路线、约束或调整结果。
- 不直接照搬展示名，迁移到业务代码后要改成路线规划语义命名。
- 不在未确认需求前大范围引入依赖或改造前端架构。

## 调用方式

后续需要前端 UI 或动效时，优先检查：

1. `react-bits/src/`：组件源码和样式。
2. `react-bits/README.md`：上游安装和使用说明。
3. `react-bits/package.json`：依赖与技术栈。

然后再按本项目现有 `frontend/` 结构迁移，保持本地 Demo 可跑。
