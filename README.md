# AI 本地路线智能规划 Agent

本项目是黑客松原型：用户输入出行或游玩目标后，系统结合 POI、约束和偏好，生成一条可以直接执行的本地路线，并支持按预算、时间、排队、拍照等条件局部调整。

当前示例场景聚焦杭州西湖周边，目标不是生成泛泛攻略，而是把多个目的地串成一条有顺序、有时间、有交通段、有调整反馈的路线。

## 当前能力

- 自然语言输入出行目标。
- 生成默认西湖半日路线。
- 展示路线总览、时间轴、交通段、地图点位和目的地卡片。
- 支持五个快捷调整：餐厅排队太久、预算降到 100、不要咖啡、只剩 2 小时、想更适合拍照。
- 支持目的地卡片的替换、删除、上移和下移。
- 后端 mock API 返回完整 `routeData`，前端通过接口消费路线数据。

## 启动方式

### 后端

```bash
cd backend
pip install -r requirements.txt
uvicorn app:app --reload
```

默认地址：

```text
http://127.0.0.1:8000
```

### 前端

```bash
cd frontend
node server.mjs
```

默认地址：

```text
http://localhost:4173
```

前端当前默认连接本地后端 API。请先启动后端，再打开前端页面。

## 文档入口

- `AGENTS.md`：项目协作规则、产品边界和工程纪律。
- `docs/api-contract.md`：第一版前后端 mock API 契约。
- `docs/routing-rules.md`：路线生成、调整、评分和产品规则主文档。
- `docs/poiCatalog字段说明.md`：后端 POI 候选池字段说明。
- `docs/routeData.sample.json`：`routeData` 返回样例。

## 当前边界

- 第一版使用 mock 数据和规则逻辑，不接真实 LLM、高德实时数据或数据库。
- 当前快捷调整是单次独立调整，暂未维护连续上下文记忆。
- 连续调整中的预算上限、禁忌类型、剩余时间、已删除节点等约束，建议在下一版由后端统一维护或由前端传入 `activeConstraints` 后交给后端合并计算。
